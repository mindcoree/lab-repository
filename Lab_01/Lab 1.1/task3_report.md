# Лабораторная работа 1.1: jQuery vs React

## Разница между императивным и декларативным подходами

### jQuery: Императивный подход

В jQuery мы используем императивный стиль программирования, где явно описываем **как** достичь результата. В Task 1 мы напрямую манипулируем DOM:

```javascript
$('#toggleButton').click(function() {
    $('#textParagraph').toggleClass('highlight');
});
```

Здесь мы вручную находим элементы и изменяем их состояние. Мы управляем DOM напрямую, указывая каждый шаг: найти элемент → добавить/удалить класс. Этот подход требует постоянного отслеживания текущего состояния DOM.

### React: Декларативный подход

React использует декларативный стиль, где мы описываем **что** должно быть отображено, а не как это сделать. В Task 2:

```javascript
const [isHighlighted, setIsHighlighted] = useState(false);

<p className={isHighlighted ? "highlight" : ""}>
  Текст
</p>
```

Мы не манипулируем DOM напрямую. Вместо этого мы описываем желаемое состояние UI в зависимости от данных. React сам определяет, какие изменения необходимо внести в DOM.

## Как работает React: State и перерисовка

React строит UI на основе концепции **Declarative UI structures** (из главы книги). Компонент описывает, как должен выглядеть интерфейс при определённом состоянии:

1. **State** — источник истины. `isHighlighted` хранит текущее состояние
2. **Render** — React вызывает функцию компонента, которая возвращает JSX
3. **Reconciliation** — React сравнивает новое виртуальное дерево со старым
4. **Update** — только изменённые части DOM обновляются

Когда вызывается `setIsHighlighted(!isHighlighted)`, React понимает, что **Data changes over time** (глава книги) и запускает перерисовку. Компонент заново выполняется с новым state, и React эффективно обновляет только изменившиеся части.

## Преимущества декларативного подхода

**Масштабируемость**: В большом приложении императивный код быстро становится сложным. С jQuery нужно отслеживать множество состояний вручную, синхронизировать изменения между компонентами, помнить, какой элемент в каком состоянии.

React решает это через однонаправленный поток данных: state → UI. Когда приложение растёт:
- Легче понять, как работает компонент (просто посмотреть на render)
- Меньше багов (нет рассинхронизации state и DOM)
- Проще тестировать (передать props/state, проверить output)
- Переиспользование компонентов без дублирования логики манипуляций DOM

Декларативный подход позволяет разработчикам думать о том, **каким должен быть** интерфейс, а не о последовательности операций для его изменения.

---

**Итого**: jQuery даёт полный контроль, но требует больше кода и внимания к деталям. React абстрагирует управление DOM, позволяя фокусироваться на бизнес-логике и состоянии приложения.
